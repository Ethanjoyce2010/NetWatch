# Security Policy

Thank you for taking the time to report a potential vulnerability in NetWatch. We take security seriously.

Preferred reporting method

- If you cannot use the GitHub security workflow, contact me privately at: <ethan.sanders10@outlook.com>

What to include

- A clear description of the vulnerability and the steps to reproduce it.
- A minimal PoC or test case if possible.
- Impact assessment and OS/versions affected.
- Your contact email for follow-up.

Policy

- Do not post security issues in public issue tracker — use the private reporting path above.
- We'll acknowledge receipt within 48 hours and aim to provide a remediation plan or timeline within 14 days.
- Credit: If you request, we will credit you in release notes once a fix is published.

Disclosure timeline

- We'll coordinate responsible disclosure. If you prefer full public disclosure immediately, explicitly state this when you report.

Thank you for helping keep NetWatch secure.
